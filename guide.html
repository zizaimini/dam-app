屏幕保护，从这里开始

了解 Dam 如何在每个场景下自动保护你的隐私，以及如何精准控制保护行为。

向下滚动
01 · 触发场景

什么时候 Dam 会介入？

Dam 持续在后台运行，监控两种核心场景——只要检测到潜在风险，立即介入。

🖥️

接入外部屏幕

连接投影仪、外接显示器、AirPlay 或 Sidecar 时，Dam 立即检测到新屏幕。如果这块屏幕从未被分类，会弹出标记界面，要求你告知这是公开还是安全环境。

物理投屏
📹

会议录屏共享

Zoom、Teams、Google Meet 等软件开始录制或共享屏幕时，Dam 通过系统 API 实时感知，弹出提示询问当前是否处于公共场景。

会议共享
⌨️

全局快捷键

任何时候，直接按下你设置的快捷键，Dam 立即执行保护动作——无需连接屏幕，无需等待。适合你预感风险、想快速清场的场景。

手动触发
🔄

已记忆的屏幕自动执行

你标记过的屏幕，Dam 会记住它的硬件 ID。下次接入同一块屏幕，直接按记忆执行策略——安全屏幕无感通过，公开屏幕立即触发保护，无需再次确认。

自动执行
02 · 标记屏幕

如何标记公开 / 安全屏幕？

每当接入未知屏幕，Dam 会在该屏幕上弹出一个全屏玻璃浮层。点击不同区域，告知 Dam 这个环境是否安全。

外部显示器 — 首次接入
🛡️
安全
点击圆圈内 → 安全
点击圆圈外 → 公开
👆 点击圆圈内或圆圈外，体验标记效果

点击圆圈内区域
标记为「安全屏幕」。Dam 不做任何保护动作，你的私人应用照常运行。此屏幕会被记住，下次接入直接通过。

点击圆圈外区域
标记为「公开屏幕」。Dam 立即触发保护——关闭或最小化你设置的敏感应用。此屏幕会被记住，下次接入自动执行。

03 · 录屏检测

开始录屏时如何选择？

Dam 实时监控屏幕共享状态。检测到录屏或会议共享时，弹出询问——由你决定是否触发保护。

Dam 检测到：系统级屏幕共享 API 触发，共享来源：Zoom。弹出浮层由你决定当前场景。

结果：什么都不发生。Dam 安静等待，继续监控。下次检测到新的共享事件，还会再次询问。

Dam 立即执行保护动作：关闭或最小化你设置的所有敏感应用。

结果：微信、银行 App、个人邮件等受保护应用立即关闭或最小化，屏幕只保留工作相关内容。

05 · 快捷键

随时一键触发保护

不需要接外部屏幕，任何时候都可以用快捷键立即保护。在任何应用中均有效。

⌃+ ⌥+ ⌘+ W

立即关闭

彻底退出所有受保护应用。适合隐私敏感度高的场景，确保内容不可见。

⌃+ ⌥+ ⌘+ Y

立即最小化

将受保护应用的所有窗口收到 Dock。会话不中断，会议结束后点击恢复。

⌥+ Tab

隐私 App 切换器

替代系统默认切换器，受保护应用不出现在列表中。投屏时切换应用更安全。

06 · 记忆机制

一次标记，终身记住

Dam 通过硬件 ID 唯一识别每块显示器，你的选择永久保存，无需重复操作。

🖥️

接入新屏幕

未知屏幕触发弹出，要求标记

🏷️

你做出选择

点击圆圈内（安全）或圆圈外（公开）

💾

策略永久存储

通过厂商 + 型号 + 序列号识别，本地存储

⚡️

下次自动执行

同一屏幕接入，直接按记忆执行，零等待

💡 想修改之前的标记？ 打开 Dam 设置 → 屏幕策略 标签页，查看所有已记录的屏幕，可以修改 安全/公开 分类、添加备注，或删除记录（删除后下次接入重新询问）。
